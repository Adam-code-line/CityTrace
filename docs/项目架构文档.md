## 1. 项目概述

CityTrace（城市足迹）是一款基于 **Flutter** 开发的移动端城市足迹记录应用。用户可以在城市中漫游时录制足迹轨迹，以"行程（Journey）"为单位组织路线，并在行程中记录"瞬间（Moment）"——包含照片、录音、文字、位置等多种媒体类型。应用还集成了 **AI 能力**，可自动生成游记、分析图片和音频内容。

### 核心业务场景

1. **足迹录制**：实时 GPS 定位，记录行走轨迹，支持行程开始/结束控制
2. **瞬间记录**：在行程中插入照片、录音、文字标注、位置等多模态内容
3. **行程管理**：对行程进行 CRUD，支持文件夹分类组织
4. **AI 增强**：AI 生成游记、图片描述、音频转写
5. **环境感知**：自动获取当前位置的地理信息（城市、区）和天气

## 5. 核心模块详解

### 5.1 入口与启动流程

**文件**: `lib/main.dart`

启动过程分为三个阶段：

```
Step 1 ── 初始化基础设施
  ├── WidgetsFlutterBinding.ensureInitialized()
  ├── StorageUtil.init()          ← 初始化 SharedPreferences
  └── MetadataUtil.init()         ← 读取应用版本号

Step 2 ── 注入全局控制器（GetX 单例，permanent: true）
  ├── Get.put(UserController())
  └── Get.put(MapTraceController())

Step 3 ── 启动应用
  ├── ScreenUtilInit             ← 屏幕适配（设计尺寸 411.4×914.3，Pixel 7）
  ├── GetMaterialApp             ← GetX 版 MaterialApp
  │   ├── theme                  ← Material3 主题（种子色蓝色）
  │   ├── initialRoute: '/'      ← 主页
  │   └── getPages               ← 路由表
  └── runApp(MainApp())
```

两个全局控制器在应用启动时即被创建并常驻内存，通过 `Get.put(..., permanent: true)` 确保不会被自动释放。

### 5.3 网络层 (core/net)

**文件**: `lib/core/net/api_client.dart`

**ApiClient** 采用 **单例模式（工厂构造）**，封装了 Dio HTTP 客户端。

#### 核心特性

1. **统一配置**
   - `baseUrl` 从 `ServerConfig.BASE_URL` 读取（当前为开发环境 Apifox 模拟地址）
   - 连接超时 / 接收超时均为 60 秒
   - Content-Type: `application/json`

2. **请求拦截器** —— 自动注入 JWT Token
   ```
   每次请求前从 StorageUtil.getToken() 读取 Token
   → 添加到 Authorization: Bearer <token> 头
   ```

3. **响应拦截器** —— 统一业务错误处理
   ```
   响应格式约定：
   {
     "code": 0,      // 0 表示成功，非 0 表示业务错误
     "msg": "...",   // 错误描述
     "data": {...}   // 业务数据
   }
   
   成功 (code == 0)  → 放行
   失败 (code != 0)  → _dispatchError(code, msg)
     ├── 全局 Toast 提示 "[code] msg"
     └── 特殊错误码 (2001/2002 = 登录失效) → 跳转登录页
   ```

4. **错误拦截器** —— 网络层面异常处理
   - 连接超时 / 连接失败 → "无法连接到服务器"
   - 接收超时 → "服务器响应超时"
   - 404 → "接口路径不存在"
   - 500 → "服务器内部错误"

5. **HTTP 方法封装**：对外暴露 `get()`、`post()`、`put()`、`patch()`、`delete()` 五个方法

### 5.7 全局控制器 (controllers)

两个全局控制器在 `main()` 中通过 `Get.put()` 注入，应用级单例，常驻内存。

#### UserController —— 用户状态管理

**职责**：管理登录/注销、用户信息缓存、全局登录态监听

**核心响应式变量**：
- `Rx<UserModel?> _user` — 当前用户信息
- `RxBool _isLoggedIn` — 登录状态

**关键方法**：

| 方法 | 触发时机 | 行为 |
|------|---------|------|
| `onInit()` | App 启动 | 加载本地缓存的用户信息 → 检查 Token 有效性 |
| `checkLoginStatus()` | 启动/恢复 | 有 Token → 请求最新 profile → 更新状态；失败 → 调用 `logout()` |
| `onLoginSuccess(user)` | 登录成功 | 保存用户信息到内存 + 本地存储 → 跳转 `/home` |
| `handleAuthLoss()` | Token 过期/失效 | 清理内存 + 本地存储 → 返回首页（可选跳转登录页） |
| `logout()` | 主动退出 | 调用后端 logout API → `handleAuthLoss()` |

#### MapTraceController —— 地图轨迹状态管理

**职责**：实时 GPS 定位、轨迹录制、行程计时

**核心响应式变量**：
- `RxList<LatLng> tracePoints` — 轨迹点列表
- `Rx<LatLng?> currentPos` — 当前坐标
- `RxBool isRecording` — 是否正在录制
- `RxBool isInJourney` — 是否在行程中
- `RxString currentJourneyId` — 当前行程 ID
- `Rx<DateTime?> startTime` — 行程开始时间
- `RxString durationStr` — 已持续时长 (HH:mm:ss)

**生命周期流程**：
```
onInit()
  └── _startLocationListening()
       └── Geolocator.getPositionStream() ← 持续监听 GPS 位置
            ├── 更新 currentPos
            └── 如果 isRecording → 追加到 tracePoints

startJourney(id, time)
  ├── 清空 tracePoints
  ├── 设置 isRecording = true, isInJourney = true
  ├── 设置 startTime
  ├── _startTimer()        ← 每秒更新 durationStr
  └── StorageUtil.setJourneyId(id)

stopJourney()
  ├── isRecording = false, isInJourney = false
  ├── 取消计时器
  ├── 重置 durationStr, startTime
  └── StorageUtil.removeJourneyId()

onClose()
  ├── 取消位置订阅
  └── 取消计时器
```

**服务端时间偏移**：`_serverTimeOffset` 支持同步服务端时间，解决本地时间与服务器时间不一致问题。

## 6. 数据流与状态管理

### 6.1 状态管理总览

CityTrace 采用 **GetX** 作为状态管理方案，遵循以下原则：

- **全局状态（应用级）**：`UserController`、`MapTraceController` → 在 `main()` 中注入 `Get.put(..., permanent: true)`
- **页面状态（页面级）**：`HomeController`、`LoginController` 等 → 在页面首次构建时通过 `Get.put()` 注入，页面销毁时自动释放
- **局部状态（Widget 级）**：通过 `.obs` 响应式变量 + `Obx()` 或 `GetBuilder` 实现细粒度刷新

### 6.2 响应式模型

```
控制器内声明：
  final RxBool _isLoggedIn = false.obs;
  final RxList<LatLng> tracePoints = <LatLng>[].obs;

对外暴露 Getter：
  bool get isLoggedIn => _isLoggedIn.value;

UI 层监听：
  Obx(() => Text(controller.isLoggedIn ? "已登录" : "未登录"))
```

### 6.3 依赖注入关系图

```
main()
  ├── Get.put(UserController())          ── 全局单例
  ├── Get.put(MapTraceController())      ── 全局单例
  └── runApp()
        └── HomePage
              └── Get.put(HomeController())
                    ├── Get.find<UserController>()
                    ├── Get.find<MapTraceController>()
                    ├── JourneyService()
                    └── ContextService()
        └── LoginPage
              └── Get.put(LoginController())
                    ├── Get.find<UserController>()
                    └── AuthService()
```

## 8. 安全设计

### 8.1 密码安全
- 客户端使用 **SHA-256** 对密码进行加盐哈希
- Salt 使用用户账号（account）—— 同一用户的 salt 恒定
- 后端应使用更安全的哈希算法（如 bcrypt）再次哈希

### 8.2 认证与会话
- 登录成功后后端返回 JWT Token
- Token 存储在本地 SharedPreferences
- 每次 API 请求通过拦截器自动添加 `Authorization: Bearer <token>` 头
- Token 过期（错误码 2001/2002）时自动清理登录态并跳转至登录页

### 8.3 权限管理
- 敏感权限（定位、相机、麦克风、相册）在使用前通过 `PermissionUtil` 统一申请
- 被永久拒绝时弹窗引导用户至系统设置手动开启
